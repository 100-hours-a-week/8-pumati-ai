# --------------------------------------------------
# 📝 Git Commit Message Style Guide
# --------------------------------------------------
#
# 커밋 메시지는 아래와 같은 형식을 따라야 합니다:
#(아래 포맷 주석 해제해서 사용하시면 됩니다)
#
# <type>: <Subject>
#
# Body (선택 사항)
#
# Footer (선택 사항: 이슈 번호 등)
#
# 예시:
#
# feat: 로그인 API 추가
#
# 로그인 API 추가, JWT 인증 구현
#
# 해결 이슈: #42
#
# --------------------------------------------------
# 🎯 Type 목록 (아래 중 하나 선택)
#
# ✨ feat:     새로운 기능 추가
# 🐛 fix:      버그 수정
# 📝 docs:     문서 변경 (README 등)
# 💄 style:    코드 포맷팅, 세미콜론 등 비기능 변경
# ♻️ refactor: 기능 변화 없는 코드 리팩토링
# ✅ test:     테스트 코드 추가 또는 수정
# 🔧 chore:    빌드 설정, 패키지 업데이트 등 기타
#
# --------------------------------------------------
# ✍️ Subject 작성 규칙
#
# - 너무 길지 않게
# - 마침표 ❌
# - 명령형 사용
#
# --------------------------------------------------
# 📚 Body 작성 규칙 (선택)
#
# - 무엇을 / 왜 변경했는지 설명
# - 어떻게는 ❌ (코드가 설명함)
# - 너무 길지 않게
#
# --------------------------------------------------
# 🏷️ Footer 작성 예시 (선택)
#
# 해결 이슈: #123
# 참고: #456, #789
#
